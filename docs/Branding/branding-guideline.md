# Vinpix Studio Branding Guideline

## Purpose

This guideline keeps Vinpix Studio communication consistent across the website, pitch pages, product pages, admin tools, decks, and outbound material.

It is based on the current code and live copy patterns in:

- `src/app/page.tsx`
- `src/app/for-business/page.tsx`
- `src/app/pitch/page.tsx`
- `src/app/layout.tsx`
- `src/components/Header.tsx`
- `src/components/WorkShowcase.tsx`

## Positioning

Vinpix Studio builds practical software with a strong product taste: internal tools, AI workflow automation, digital products, and games.

For business audiences, Vinpix Studio should be positioned as a delivery partner that turns manual workflows into working tools quickly, without bloated outsourcing scopes.

For product and game audiences, Vinpix should feel independent, sharp, playful, and hands-on.

## Brand Pillars

| Pillar | Meaning | Use In Copy |
| --- | --- | --- |
| Practical speed | Ship useful pilots in weeks, not months. | "Pilot in 2-4 weeks", "rapid pilots", "working internal tools" |
| Product taste | Build tools that teams can actually use. | "clean UI", "functional prototype", "source code handoff" |
| Automation clarity | Start with one painful workflow and define ROI. | "define the bottleneck", "measurable ROI", "manual ops" |
| Independent craft | Small studio energy with full ownership. | "independent studio", "designed and built", "zero compromise" |
| Playful edge | Games and product copy can be witty without becoming vague. | "pure fun", "co-op chaos", direct one-liners |

## Voice

Vinpix Studio should sound:

- Direct
- Confident
- Builder-led
- Sharp but approachable
- Practical before poetic
- Slightly playful when talking about games or personal projects

Vinpix Studio should not sound:

- Corporate and vague
- Overly luxurious
- Hype-driven
- Crypto-first when explaining games
- Like a generic agency template

## Tone By Context

| Context | Tone | Example Pattern |
| --- | --- | --- |
| Homepage | Independent, broad, portfolio-led | "Independent software studio for products, games, and AI-powered workflows." |
| B2B page | Clear, ROI-focused, trustworthy | "We automate manual workflows in 2-4 weeks." |
| Case studies | Outcome-first | "Eliminated 10+ hours/week of manual tracking." |
| Games | Punchy, playful, fun-first | "Co-op chaos. Yell at your friends..." |
| Support/legal | Simple and plain | "If you need help, contact..." |
| Admin/tools | Functional and concise | Use clear action labels; avoid brand-heavy prose. |

## Messaging Rules

### Preferred Words

- automate
- workflow
- pilot
- internal tools
- AI-powered workflows
- delivery partner
- measurable ROI
- source code handoff
- practical
- functional prototype
- product
- games
- independent studio

### Avoid

- "full-service digital transformation" unless a page truly explains that scope
- "AI magic"
- "revolutionary"
- "seamless synergy"
- "world-class" without evidence
- Web3 jargon before the gameplay value is clear
- Long abstract mission statements

## Visual Identity

The current site uses a stark editorial/technical style:

| Element | Guideline |
| --- | --- |
| Primary background | Off-white `#F0F0F0` |
| Primary text | Black |
| High-contrast sections | Black background with white text |
| Accent usage | Minimal; use red/blue only for emphasis or product-specific contrast |
| Borders | Strong black borders, usually 2px |
| Cards | Rectangular, sharp, utility-first; avoid soft decorative cards |
| Motion | Reveal animations, dither texture, subtle interactive effects |
| Imagery | Product/game/project images should be real and inspectable; grayscale hover is acceptable |

## Typography

| Usage | Style |
| --- | --- |
| Hero headings | Very large, black, uppercase, tight tracking |
| Section headings | Bold uppercase |
| Supporting copy | Medium weight, direct, readable |
| Utility labels | Small uppercase, wide tracking |

- Lexend for sans text

## Logo And Name Usage

Use `Vinpix Studio` in formal contexts:

- Website header
- Legal/footer copy
- Business pages
- Decks
- Client-facing proposals

Use `Vinpix` when the context is already clear:

- Homepage hero
- Short labels
- Informal product or portfolio references

Do not write:

- `VinPix`
- `VINPIX Studio`
- `Vinpix Studios`
- `Vinpix AI Studio`

## B2B Messaging Framework

Use this order for business pages, proposals, and decks:

1. Name the bottleneck.
2. Explain the workflow to automate.
3. Define the pilot scope.
4. Show what gets built.
5. State handoff, hosting, ownership, and documentation terms.
6. Close with a direct CTA.

## Writing Examples

### Good

> We start with one painful manual workflow, define the expected ROI, and build a working pilot in 2-4 weeks.

Why it works: specific, scoped, outcome-oriented.

### Good

> Independent software studio for products, games, and AI-powered workflows.

Why it works: short, broad, accurate to the homepage.

### Avoid

> We revolutionize digital transformation through next-generation AI solutions.

Why it fails: generic, unsourced, and not aligned with the current brand voice.

## Implementation Notes

When updating brand-related UI:

- Keep black/off-white contrast as the default system.
- Prefer bold type, clear grid structure, and hard borders.
- Keep copy short and operational.
- Preserve the distinction between studio brand, B2B offer, and game/product pages.
- Check `src/app/page.tsx`, `src/app/for-business/page.tsx`, and `src/components/Header.tsx` before changing homepage or business messaging.
